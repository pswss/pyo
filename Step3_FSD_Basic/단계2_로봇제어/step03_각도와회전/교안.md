## 1. 도입 (10분)

### 질문으로 시작하기
> "시계 초침이 한 바퀴 돌면 몇 도? 그걸 라디안으로 표현하면?"

**도(degree)**: 우리에게 익숙한 단위. 한 바퀴 = 360°  
**라디안(radian)**: 수학/컴퓨터가 좋아하는 단위. 한 바퀴 = 2π ≈ 6.28

### 왜 컴퓨터는 라디안을 쓸까?
```python
import math

# sin, cos 등 수학 함수는 라디안 단위를 사용!
math.sin(math.pi / 2)  # 90도 = π/2 라디안 → 결과: 1.0
math.sin(90)           # 이건 90 라디안! 완전히 다른 값
```

### 변환 공식
```
도 → 라디안: radians = degrees × π / 180
라디안 → 도: degrees = radians × 180 / π
```

## 2. Angle 클래스 분석 (15분)

### 우리 프로젝트의 Angle 클래스
[src/data_structures/angle.py](../../../src/data_structures/angle.py)

```python
class Angle:
    RADIANS = 0
    DEGREES = 1

    def __init__(self, value, unit=RADIANS):
        if unit == self.RADIANS:
            self.__radians = float(value)
        else:
            self.degrees = value  # setter 호출
```

사용 방법:
```python
a1 = Angle(3.14)              # 라디안으로 생성 (≈ 180도)
a2 = Angle(90, Angle.DEGREES) # 도로 생성 (90도)

print(a1.degrees)   # → 180.0
print(a2.radians)   # → 1.5707...
```

### normalize() — 각도 정규화

**문제**: 로봇이 계속 회전하면 각도가 무한히 커집니다!
- 540도 → 실제로는 180도와 같음
- -90도 → 실제로는 270도와 같음

**해결**: 모든 각도를 0 ~ 2π (0° ~ 360°) 범위로 맞추기

```python
def normalize(self):
    self.__radians %= 2 * math.pi   # 0 ~ 2π 범위로

    if self.__radians < 0:
        self.__radians += 2 * math.pi  # 음수 처리
```

## 3. 우리가 고친 버그 복습! (10분)

### 버그 발견 당시 코드 (틀린 코드)
```python
if self.__radians < 0:
    self.__radians += 2 + math.pi   # ← 틀림! + 가 아니라 *
```

### 이게 왜 문제였을까?

```python
import math

angle = -1.0  # -1 라디안 (약 -57도)

# 틀린 코드:
wrong = angle + (2 + math.pi)   # = -1 + 5.14 = 4.14 라디안 (237도)

# 맞는 코드:
correct = angle + (2 * math.pi) # = -1 + 6.28 = 5.28 라디안 (302도)

# 실제로 -1 라디안 = 약 302도가 맞음!
```

**결과**: 모든 회전 계산에서 약 **40도 오차** 발생!  
→ 로봇이 엉뚱한 방향을 바라보고 이동함

## 4. 방향 계산 원리 (10분)

### atan2 — 두 점 사이의 방향 계산

```python
import math

# 나는 (0, 0)에 있고, 목표는 (1, 1)에 있다
my_x, my_y   = 0, 0
goal_x, goal_y = 1, 1

dx = goal_x - my_x  # 1
dy = goal_y - my_y  # 1

angle = math.atan2(dy, dx)  # π/4 라디안 = 45도
print(f"{math.degrees(angle):.1f}도")  # 45.0도
```

### 시각적으로 이해하기
```
          목표
           ↗ (45도)
          /
    나 →─/
```

## 핵심 정리

```
도 ↔ 라디안: degrees × π/180 ↔ radians × 180/π
정규화: 모든 각도를 0 ~ 2π (0° ~ 360°) 범위로
방향 계산: math.atan2(dy, dx) 사용
버그 주의: 2 + π ≠ 2 × π (반드시 곱셈!)
```
