## 2. 프로그램 시작점 읽기 (10분)

[src/main.py](../../../src/main.py)

```python
from executor.executor import Executor
from mapping.mapper import Mapper
from robot.robot import Robot

def main():
    robot = Robot(time_step=32)           # 로봇 생성 (32ms 간격)
    mapper = Mapper(...)                  # 지도 시스템 생성
    executor = Executor(mapper, robot)    # 전체 흐름 관리자 생성

    executor.run()                        # 루프 시작!

main()
```

### 핵심 질문

> **"time_step=32가 뭔가요?"**
>
> 시뮬레이션이 32밀리초(0.032초)마다 한 번 업데이트됩니다.
> 1초 = 약 31번 실행. 8분 경기 = 약 15,000번 실행.

## 3. 세 개의 핵심 클래스 (15분)

### Robot — "몸"

```
Robot
├── GPS        → 현재 위치 (x, y 좌표)
├── Gyroscope  → 현재 방향 (각도)
├── Lidar      → 주변 벽 거리 (360도 스캔)
├── Camera     → 피해자 이미지
└── DriveBase  → 바퀴 제어
```

**비유**: 로봇의 신체 + 감각기관

---

### Mapper — "기억"

```
Mapper
├── WallMapper      → 벽 위치를 격자에 표시
├── FloorMapper     → 늪/체크포인트 표시
└── FixtureMapper   → 발견한 피해자 위치 기억
```

**비유**: 탐험가가 그리는 지도

---

### Executor — "두뇌"

```
Executor
├── StateMachine    → 지금 뭐하는 중? (탐색/보고/복귀...)
├── Agent           → 다음 목표 좌표는?
└── FixtureClassifier → 카메라 이미지가 피해자인가?
```

**비유**: 지휘관 — 몸(Robot)에게 명령, 지도(Mapper)를 보고 결정

## 4. 실제 흐름 한 사이클 (15분)

```
매 32ms마다:
┌─────────────────────────────────────────────────────┐
│  robot.update()                                     │
│    └─ GPS, 자이로, 라이다, 카메라 값 읽기            │
│                                                     │
│  mapper.update()                                    │
│    └─ 라이다 데이터로 벽 기록                        │
│    └─ 카메라로 바닥/피해자 기록                      │
│                                                     │
│  state_machine.run()         ← 현재 상태 실행       │
│    ├─ [탐색 중] → agent.update() → 목표 좌표 받기   │
│    │              robot.move_to_coords() 실행        │
│    ├─ [보고 중] → 피해자 신고 신호 전송              │
│    └─ [복귀 중] → 출발점으로 이동                   │
└─────────────────────────────────────────────────────┘
```

## 5. flags.py — 조절 패널 (5분)

[src/flags.py](../../../src/flags.py)

```python
SHOW_DEBUG = False    # True로 바꾸면 콘솔에 위치 출력
DO_SLOW_DOWN = False  # True로 바꾸면 시뮬레이션 느려짐 (디버그용)
SLOW_DOWN_S = 0.1    # 느리게 할 때 딜레이 초
TUNE_FILTER = False  # 색상 필터 튜닝 창 열기
```

> 디버그할 때 `SHOW_DEBUG = True`로 바꿔 두면
> 매 스텝마다 위치와 방향이 출력됩니다.

## 핵심 키워드

| 키워드 | 의미 |
|--------|------|
| `time_step` | 시뮬레이션 갱신 간격 (밀리초) |
| `do_loop()` | 한 스텝 진행 + 계속할지 반환 |
| `StateMachine` | 현재 행동 상태 관리 |
| `Agent` | 다음 목표 위치 결정 |
| `compiled.py` | 대회용 단일 파일 (수정 금지!) |
