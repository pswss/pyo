# Step 03 — 데이터 흐름: 센서에서 결정까지

## 학습 목표
- 센서 데이터가 어떻게 가공되어 로봇의 행동이 되는지 이해한다
- 각 클래스가 어떤 데이터를 받고 어떤 데이터를 내보내는지 안다
- 실제 코드에서 데이터가 흐르는 경로를 추적할 수 있다

---

## 1. 전체 데이터 흐름 (10분)

```
[물리 세계 / 시뮬레이터]
         │
         │ 라이다 반사, GPS 신호, 카메라 빛, 자이로 각속도
         ▼
┌─────────────────────────────────────────────────┐
│  Robot.update()                                 │
│                                                 │
│  GPS      → position (x, y) 미터 단위           │
│  Gyroscope → orientation (각도, 라디안)          │
│  Lidar    → point_cloud (거리 값 배열)           │
│  Camera   → image (numpy 배열, RGB)             │
└───────────────────────┬─────────────────────────┘
                        │
         ┌──────────────┼──────────────┐
         ▼              ▼              ▼
  [Mapper]        [FixtureClassifier] [Agent]
  라이다→격자지도   이미지→피해자유무   지도→목표좌표
         │                │                │
         ▼                ▼                ▼
  벽 위치 기록      "H" / None        (x, y) 좌표
                         │
                         ▼
               [Executor/StateMachine]
               → 탐색 계속 / 피해자 신고
                         │
                         ▼
               [DriveBase]
               왼쪽 바퀴 속도, 오른쪽 바퀴 속도
                         │
                         ▼
            [물리 엔진 → 로봇 실제 이동]
```

## 2. 센서별 데이터 형태 (20분)

### GPS → position

[src/robot/devices/gps.py](../../../src/robot/devices/gps.py)

```python
# GPS가 반환하는 값
position = Position2D(x=0.12, y=-0.36)  # 미터 단위

# 사용 예
print(robot.position.x)  # 0.12
print(robot.position.y)  # -0.36
```

**특징**:
- Webots 좌표계: X=오른쪽, Y=아래쪽 (수학 좌표와 Y축 반전!)
- 출발점이 (0, 0)이 아닐 수 있음 → `position_offsets`로 보정

---

### Gyroscope → orientation

[src/robot/devices/gyroscope.py](../../../src/robot/devices/gyroscope.py)

```python
# 방향이 Angle 객체로 반환
orientation = robot.orientation       # Angle 객체
print(orientation.radians)           # 1.5707... (라디안)
print(orientation.degrees)           # 90.0 (도)
```

**특징**:
- 자이로스코프는 각속도를 측정 → 적분해서 각도 계산
- GPS 이동 방향으로 보정 (빠르게 움직일 때만 정확)
- `auto_decide_orientation_sensor`: 늪 근처에서는 자이로만 사용

---

### Lidar → point_cloud

[src/robot/devices/lidar.py](../../../src/robot/devices/lidar.py)

```python
# 라이다: 360도 방향으로 거리 측정
point_cloud = robot.get_point_cloud()
# [(x1,y1,z1), (x2,y2,z2), ...]  실제 좌표들의 리스트
# 벽에 닿은 지점들

out_of_bounds = robot.get_out_of_bounds_point_cloud()
# 측정 범위 밖 = 열린 공간
```

**특징**:
- 2층 레이어 중 2번 레이어(수평)만 사용: `layers_used=(2,)`
- 6 time_step 마다 갱신 (매 192ms)
- Mapper가 이 데이터로 격자 지도에 벽을 표시

### Camera → image

[src/robot/devices/camera.py](../../../src/robot/devices/camera.py)

```python
cam_images = robot.get_camera_images()
for cam in cam_images:
    img = cam.image      # numpy 배열 (H, W, 3) BGR 형식
```

**특징**:

- 3 time_step 마다 갱신 (매 96ms)
- FixtureClassifier가 이 이미지로 피해자 감지

## 3. 데이터 가공: Mapper (15분)

[src/mapping/mapper.py](../../../src/mapping/mapper.py)

```python
# Mapper.update()가 받는 입력
mapper.update(
    point_cloud       = [(x,y,z), ...],   # 라이다: 벽 위치
    out_of_bounds_pc  = [(x,y,z), ...],   # 라이다: 빈 공간
    lidar_detections  = [...],            # 라이다: 특수 감지
    camera_images     = [CamImage, ...],  # 카메라: 이미지들
    robot_position    = Position2D,       # GPS: 로봇 위치
    robot_orientation = Angle,            # 자이로: 로봇 방향
    time              = float,            # 경과 시간(초)
)
```

### 내부에서 일어나는 일

```
point_cloud 좌표들
    │
    ▼
PixelGrid (numpy 2D 배열)
    ├── 벽 = True
    ├── 빈공간 = False
    └── 미탐색 = 특별 값
    │
    ▼
ConvolutionFilter (로봇 크기만큼 팽창)
    └── 로봇 중심이 지나갈 수 있는 영역 계산
```

## 4. 데이터 가공: Agent (15분)

[src/agent/agent.py](../../../src/agent/agent.py)

```python
# Agent.update()가 하는 일
def update(self):
    # 1. 어느 서브에이전트가 실행 가능한지 확인
    for subagent in [GoToFixtures, FollowWalls, GoToNonDiscovered]:
        if subagent.is_possible():      # 실행 조건 충족?
            self.current_agent = subagent
            break
    
    # 2. 선택된 에이전트가 목표 좌표 계산
    self.current_agent.update()

# Agent.get_target_position()이 반환하는 값
target = agent.get_target_position()  # Position2D(x, y)
```

### 서브에이전트별 목표 결정 방식

| 서브에이전트 | 입력 | 출력 | 알고리즘 |
|-------------|------|------|---------|
| GoToFixtures | 감지된 피해자 위치 | 피해자 좌표 | 가장 가까운 피해자 |
| FollowWalls | 현재 지도 | 벽 근처 좌표 | 벽에서 일정 거리 유지 |
| GoToNonDiscovered | 현재 지도 | 미탐색 좌표 | BFS로 가장 가까운 미탐색 |

## 6. 실제 코드에서 데이터 추적하기 (10분)

### 피해자 발견 → 신고까지 데이터 경로

```python
# 1. 카메라 이미지 읽기 (executor.py:226)
cam_images = self.robot.get_camera_images()

# 2. 피해자 감지 (executor.py:231)
fixtures = self.fixture_detector.find_fixtures(cam_image.image)

# 3. 피해자 분류 (executor.py:232)
self.letter_to_report = self.fixture_detector.classify_fixture(fixtures[0])
# → "H", "S", "U", "위험물" 중 하나

# 4. 방향 기록 (executor.py:233)
self.report_orientation = cam_image.data.horizontal_orientation
# → 피해자가 있는 방향 (각도)

# 5. 상태 전환 (executor.py:238)
change_state_function("report_fixture")

# 6. 피해자 쪽 회전 (executor.py:319)
self.seq_rotate_to_angle(self.report_orientation.degrees)

# 7. 신호 전송 (executor.py:354)
self.robot.comunicator.send_victim(self.robot.raw_position, self.letter_to_report)
```

## 핵심 정리

```
센서 데이터 형식:
  GPS         → Position2D(x, y)   [미터]
  Gyroscope   → Angle(radians)     [라디안]
  Lidar       → [(x,y,z), ...]     [미터 좌표 리스트]
  Camera      → np.ndarray (H,W,3) [BGR 이미지]

가공 결과:
  Mapper      → PixelGrid (2D bool 배열, 벽/빈공간/미탐색)
  Agent       → Position2D (다음 목표 좌표)
  Classifier  → str or None ("H"/"S"/"U"/None)

최종 출력:
  DriveBase   → (left_vel, right_vel) [-1.0 ~ 1.0]
```
