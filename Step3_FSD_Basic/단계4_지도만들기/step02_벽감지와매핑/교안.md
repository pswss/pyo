# Step 02 — 벽 감지와 매핑

## 학습 목표
- 라이다 포인트 → 격자 지도 변환 과정을 이해한다
- "노이즈 필터링"이 왜 필요한지 안다
- 로봇 크기를 고려한 이동 가능 영역 계산을 이해한다

---

## 1. 도입 (10분)

### 라이다 → 지도 변환 흐름

```
라이다 포인트 (상대 좌표)
        ↓  로봇 위치 더하기 (절대 좌표로)
절대 좌표
        ↓  좌표 → 격자 인덱스 변환
격자 인덱스
        ↓  해당 칸을 "벽" 으로 표시
격자 지도 업데이트
```

### 왜 한 번만으로 확정하면 안 될까?

라이다는 가끔 **잘못된 값**을 반환합니다 (노이즈)!
- 유리, 거울에 빛이 튕겨나감
- 센서 오차
- 빠른 이동 중 떨림

**해결**: 같은 위치가 **3번 이상** 감지되면 벽으로 확정!

### 포인트 클라우드 → 격자 업데이트

```python
def load_in_bounds_point_cloud(self, point_cloud, robot_position):
    for p in point_cloud:
        # 1. 절대 좌표 계산 (로봇 위치 + 상대 좌표)
        point = np.array(p) + robot_position

        # 2. 격자 인덱스로 변환
        point_grid_index = self.grid.coordinates_to_grid_index(point)

        # 3. 지도 확장 (필요하면)
        self.grid.expand_to_grid_index(point_grid_index)

        # 4. 해당 격자에 감지 카운트 추가
        self.occupy_point(point_array_index)
```

### 노이즈 필터링 (3번 이상 감지 → 벽 확정)

```python
def occupy_point(self, point_array_index):
    # 감지 카운트 증가
    self.grid.arrays["detected_points"][row, col] += 1

    # 임계값(3) 이상이면 벽으로 확정
    if self.grid.arrays["detected_points"][row, col] > self.to_boolean_threshold:
        self.grid.arrays["walls"][row, col] = True
```
