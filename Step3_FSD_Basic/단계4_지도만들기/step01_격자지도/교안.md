## 1. 도입 (10분)

### 왜 격자 지도를 쓸까?

로봇이 미로를 탐색하려면 "어디가 벽이고 어디가 길인지" 기억해야 합니다.

**방법 1**: 벽의 정확한 좌표 저장 → 복잡하고 느림  
**방법 2**: 격자로 나눠서 True/False로 저장 → **단순하고 빠름!**

```
실제 미로             격자 지도 (True=벽, False=길)
                      
  ████████            [[T, T, T, T, T, T, T, T],
  █      █             [T, F, F, F, F, F, F, T],
  █ ██   █             [T, F, T, T, F, F, F, T],
  █      █             [T, F, F, F, F, F, F, T],
  ████████             [T, T, T, T, T, T, T, T]]
```

## 2. numpy 배열 = 지도 (15분)

```python
import numpy as np

# 5×8 격자 지도 생성 (모두 False = 이동 가능)
grid = np.zeros((5, 8), dtype=bool)

# 벽 표시
grid[0, :] = True   # 위 벽 전체
grid[4, :] = True   # 아래 벽 전체
grid[:, 0] = True   # 왼쪽 벽 전체
grid[:, 7] = True   # 오른쪽 벽 전체

# 특정 위치 확인
print(grid[2, 3])   # False → 이동 가능
print(grid[0, 0])   # True  → 벽
```

### 배열 인덱스 vs 좌표

주의! numpy 배열은 **[행, 열]** = **[y, x]** 순서!

```python
grid[행(y), 열(x)]
grid[2, 5]  → y=2, x=5 위치
```

## 3. 실제 좌표 ↔ 격자 인덱스 변환 (15분)

### 실제 세계에서 격자로
```
실제 크기: 2m × 2m
격자 해상도: 10px/m (1격자 = 10cm)
→ 격자 크기: 20×20

변환 공식:
  격자_x = int(실제_x × 해상도)
  격자_y = int(실제_y × 해상도)
```

예시:
```python
resolution = 10  # 10격자 = 1미터

real_x, real_y = 0.5, 1.2   # 실제 좌표 (미터)
grid_col = int(real_x * resolution)  # 5
grid_row = int(real_y * resolution)  # 12
```

### 실제 프로젝트 코드
[src/data_structures/compound_pixel_grid.py](../../../src/data_structures/compound_pixel_grid.py)

```python
def coordinates_to_grid_index(self, coordinates):
    """실제 좌표(미터) → 격자 인덱스"""
    return (np.array(coordinates) * self.resolution).astype(int)
```

## 4. 동적 지도 확장 (10분)

**문제**: 미로 크기를 미리 모른다!  
**해결**: 로봇이 탐색하면서 지도를 **자동으로 늘린다**

```
탐색 시작: 10×10 격자
     ↓ 로봇이 오른쪽으로 이동
10×15 격자로 확장
     ↓ 로봇이 위로 이동
15×15 격자로 확장
```
