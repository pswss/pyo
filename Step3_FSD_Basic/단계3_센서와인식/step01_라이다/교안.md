## 1. 도입 (10분)

### 라이다란?
**Li**ght **D**etection **A**nd **R**anging

> "레이저를 쏘고, 반사되어 돌아오는 시간으로 거리를 잰다"

```
                 ╔═══╗  ← 벽
  레이저 →→→→→ ║   ║
  로봇 ●         ║   ║
  ←←←←←← 반사  ╚═══╝

  거리 = 빛의 속도 × 왕복 시간 / 2
```

우리 로봇의 라이다는 **360도** 회전하며 주변 모든 방향을 스캔합니다!

### 실생활 예시
- 자율주행차의 지붕에 있는 돌아가는 센서
- 스마트폰 뒤 카메라의 LiDAR 센서 (아이폰 Pro)

## 2. 포인트 클라우드 이해 (15분)

### 포인트 클라우드란?
라이다가 한 번 스캔하면 수백 개의 거리 측정값을 반환합니다.  
각 측정값 = 하나의 **점(point)** → 모이면 **구름(cloud)**처럼 보임

```
           ▓▓▓▓▓▓▓▓  ← 벽
        ·  ·  ·  ·  · ← 라이다 포인트들
      ·                  ·
    ·                      ·
   ·          ●             · ← 로봇
    ·                      ·
      ·                  ·
        ·  ·  ·  ·  · ← 바닥 포인트
           ▓▓▓▓▓▓▓▓  ← 벽
```

### 실제 코드 분석
[src/robot/devices/lidar.py](../../../src/robot/devices/lidar.py)

```python
def get_point_cloud(self):
    """라이다 포인트 클라우드를 반환 (로봇 기준 상대 좌표)"""
    return self.__point_cloud

# 반환값: [(x1, y1), (x2, y2), ...] 형태의 리스트
# x, y: 로봇 중심으로부터의 상대 거리 (미터)
```

## 3. 라이다 설정 분석 (10분)

[src/robot/robot.py](../../../src/robot/robot.py) — 42번 줄

```python
lidar_interval = 6  # 6스텝마다 한 번씩 업데이트
self.lidar = Lidar(
    webots_device = self.robot.getDevice("lidar"),
    time_step     = self.time_step * lidar_interval,
    step_counter  = StepCounter(lidar_interval),
    layers_used   = (2,)   # 레이어 2번만 사용 (수평 방향)
)
```

**layers_used = (2,)** 의 의미:
- 라이다는 여러 층(layer)의 레이저를 쏩니다
- 층 0: 약간 아래 방향 → 바닥/홀 감지
- 층 2: 수평 방향 → **벽 감지에 주로 사용**
