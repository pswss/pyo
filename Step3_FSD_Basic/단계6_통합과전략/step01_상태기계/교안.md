## 1. 도입 (10분)

### 상태 기계란?

> "로봇은 어떤 **상태**에 있고, 특정 조건이 되면 **다른 상태로 전환**된다"

우리 실생활 예시:
```
신호등: 빨강 → (30초) → 초록 → (30초) → 노랑 → (5초) → 빨강
엘리베이터: 대기 → 이동중 → 문열림 → 대기
```

### 로봇의 상태들

```
                 피해자 발견
    탐색 ─────────────────→ 피해자 보고
      ↑                          │
      └──────────────────────────┘
             보고 완료

    탐색 ──→ 시간 종료 ──→ 출발점 복귀
    탐색 ──→ 갇힘 감지 ──→ 탈출 시도
```

## 2. 실제 코드 분석 (20분)

[src/flow_control/state_machine.py](../../../src/flow_control/state_machine.py)

### StateMachine 클래스

```python
class StateMachine:
    def __init__(self, initial_state, ...):
        self.state = initial_state
        self.state_functions  = {}    # 상태별 실행 함수
        self.allowed_changes  = {}    # 허용된 상태 전환
```

### 상태 등록

```python
# Executor에서 상태 등록
sm.create_state(
    name     = "explore",
    function = self.state_explore,          # 탐색 상태 실행 함수
    possible_changes = {"report_fixture",   # 전환 가능한 상태들
                        "return_to_start",
                        "stuck"}
)
```

### 상태 실행 및 전환

```python
# 매 스텝 실행
sm.run()   # → 현재 상태의 함수 실행

# 상태 전환
def state_explore(change_state):
    if 피해자_발견:
        change_state("report_fixture")  # 상태 전환!
```

## 3. Executor 전체 흐름 (15분)

[src/executor/executor.py](../../../src/executor/executor.py)

```
main()
  ↓
robot.do_loop() (매 32ms)
  ↓
robot.update()    ← 센서 갱신
mapper.update()   ← 지도 갱신
state_machine.run() ← 현재 상태 실행

┌─────────────────────────────────────┐
│ 상태: explore (탐색)                 │
│   → agent.update() 실행             │
│   → 경로 따라 이동                   │
│   → 피해자 감지 체크                 │
└─────────────────────────────────────┘
      ↓ 피해자 발견!
┌─────────────────────────────────────┐
│ 상태: report_fixture (보고)          │
│   → 정지                            │
│   → 심판 시스템에 신호 전송           │
│   → 3초 대기                        │
└─────────────────────────────────────┘
      ↓ 보고 완료
┌─────────────────────────────────────┐
│ 상태: explore (탐색 재개)            │
└─────────────────────────────────────┘
```
