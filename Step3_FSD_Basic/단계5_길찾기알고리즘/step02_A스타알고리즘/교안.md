## 1. BFS의 한계 (10분)

### BFS는 무조건 모든 방향을 탐색

```
목표가 오른쪽에 있어도...
BFS는 왼쪽, 위, 아래도 모두 탐색!

S = 출발, G = 목표 (오른쪽 끝)

BFS 탐색 순서:
← ← ← S → → → → G
BFS는 왼쪽도 다 탐색함 (낭비!)
```

### A*는 목표 방향으로 우선 탐색

```
"G가 오른쪽에 있으니 오른쪽 먼저!"

A* 탐색 순서:
        S → → → → G
거의 직선으로 탐색 (효율적!)
```

## 2. A* 핵심 공식 (15분)

```
f(n) = g(n) + h(n)

g(n) = 출발점에서 n까지의 실제 비용 (이미 간 거리)
h(n) = n에서 목표까지의 예상 비용 (휴리스틱)
f(n) = 전체 예상 비용 (작을수록 좋은 경로)
```

### 휴리스틱 — 목표까지의 "대략적인 거리 추정"

```python
# 실제 프로젝트 코드
# src/algorithms/np_bool_array/efficient_a_star.py — 38번 줄

@staticmethod
def heuristic(start, target):
    dy = abs(start[0] - target[0])
    dx = abs(start[1] - target[1])
    return min(dx, dy) * 15 + abs(dx - dy) * 10
```

**왜 이런 수식?**
- 대각선 이동이 없으므로 맨해튼 거리 변형
- 대각선 방향: `min(dx, dy) * 15` (가로+세로 한 번에)
- 나머지 직선: `abs(dx - dy) * 10`

## 3. 우선순위 큐 (힙) — A*의 핵심 자료구조 (10분)

BFS: 큐(먼저 들어간 것이 먼저 나옴)  
A*: **우선순위 큐** (f값이 작은 것이 먼저 나옴)

```python
from heapq import heappush, heappop

pq = []
heappush(pq, (5, "노드A"))   # f=5
heappush(pq, (2, "노드B"))   # f=2
heappush(pq, (8, "노드C"))   # f=8

print(heappop(pq))  # (2, "노드B") → f가 가장 작은 것부터!
print(heappop(pq))  # (5, "노드A")
print(heappop(pq))  # (8, "노드C")
```

```python
        if node.location == end_node.location:

            child.g = node.g + 1             # 실제 비용 +1
            child.h = heuristic(child, end)  # 예상 비용
            child.p = 선호도_비용             # 벽 근처 패널티

            heappush(openList, child)
```

### p (preference) 란?

벽 근처로 가면 추가 비용 부여 → 로봇이 벽에서 멀리 이동!
