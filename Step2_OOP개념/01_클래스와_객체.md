## 1. 왜 클래스가 필요한가?

함수만 쓸 때의 문제:

```python
# 함수만으로 로봇 두 대를 관리하면?
robot1_x = 0.0
robot1_y = 0.0
robot1_speed = 1.0

robot2_x = 1.5
robot2_y = 0.5
robot2_speed = 0.8

def move_robot(x, y, speed, direction):
    ...  # 어느 로봇인지 구분하려면 변수를 계속 넘겨야 함
```

클래스를 쓰면:

```python
class Robot:
    def __init__(self, x, y, speed):
        self.x = x
        self.y = y
        self.speed = speed

    def move(self, direction):
        ...  # self 안에 이 로봇만의 데이터가 있음

robot1 = Robot(0.0, 0.0, 1.0)
robot2 = Robot(1.5, 0.5, 0.8)

robot1.move("forward")   # robot1의 데이터만 사용
robot2.move("left")      # robot2의 데이터만 사용
```

**클래스 = 데이터 + 함수를 하나로 묶은 설계도**
**인스턴스 = 설계도로 만든 실제 로봇**

## 2. 클래스 기본 구조

```python
class 클래스이름:               # 1. class 키워드 + 이름 (대문자로 시작)

    def __init__(self, ...):   # 2. 생성자: 인스턴스를 만들 때 자동 실행
        self.변수 = 값          # 3. 인스턴스 변수: 이 객체만의 데이터

    def 메서드이름(self, ...):  # 4. 메서드: 클래스 안의 함수
        ...                    #    첫 번째 매개변수는 항상 self
```

### `self`란?

`self`는 **"나 자신"** 을 가리키는 변수입니다.

```python
class Sensor:
    def __init__(self, name, max_range):
        self.name = name          # 이 센서의 이름
        self.max_range = max_range

    def read(self, distance):
        if distance > self.max_range:   # 이 센서의 최대 범위와 비교
            return None
        return distance

lidar = Sensor("lidar", 2.0)    # Sensor 인스턴스 생성
camera = Sensor("camera", 0.5)

print(lidar.read(1.5))    # 1.5  (범위 안)
print(camera.read(1.5))   # None (범위 밖)
```

`lidar.read(1.5)` 를 호출하면 파이썬이 내부적으로 `Sensor.read(lidar, 1.5)` 로 변환합니다.
그래서 `self` = `lidar` 가 됩니다.

## 3. 실제 소스와 비교

`src/rescue_robot.py` 의 `RescueRobot` 클래스:

```python
class RescueRobot:
    def __init__(self):                     # 인자 없이 생성
        self._robot  = Robot(time_step=32)  # 내부에서 다른 객체를 만들어 저장
        self._mapper = Mapper(...)
        self._executor = Executor(...)

    def stop(self):                         # 메서드
        self._robot.move_wheels(0, 0)       # self._robot 의 메서드를 호출

    def move_forward(self, speed=1.0):
        self._robot.move_wheels(speed, speed)
```

```python

robot.move_forward()    # 메서드 호출
robot.stop()
```

## 4. 인스턴스 변수 vs 클래스 변수

```python
class Robot:
    count = 0             # 클래스 변수: 모든 인스턴스가 공유

    def __init__(self, name):
        self.name = name  # 인스턴스 변수: 각 객체마다 다름
        Robot.count += 1  # 클래스 변수 수정

r1 = Robot("로봇1")
r2 = Robot("로봇2")

print(r1.name)      # 로봇1
print(r2.name)      # 로봇2
print(Robot.count)  # 2  (두 번 만들었으므로)
```

소스 코드에서: `src/agent/agent.py`
```python
self.end_reached_distance_threshold = 0.04   # 인스턴스 변수 (각 에이전트마다 설정 가능)
self.max_time = 8 * 60                       # 인스턴스 변수 (8분 = 480초)
```

## 5. 메서드의 종류

```python
class DriveBase:
    def __init__(self, max_speed):
        self.max_speed = max_speed
        self._current_speed = 0.0

    def set_speed(self, left, right):              # 일반 메서드
        self._current_speed = (left + right) / 2

    def get_average_speed(self):                   # 일반 메서드
        return self._current_speed

    def stop(self):
        self.set_speed(0, 0)                       # 메서드 안에서 다른 메서드 호출 가능
```

## 🔍 더 읽어보기

소스 코드에서 클래스가 어떻게 연결되는지 구조도:

```
RescueRobot
  ├── Robot            (src/robot/robot.py)
  │     └── DriveBase, Cameras, Lidar, GPS ...
  ├── Mapper           (src/mapping/mapper.py)
  │     └── FixtureMapper, RobotMapper ...
  └── Executor         (src/executor/executor.py)
        ├── Agent
        ├── StateMachine
        └── FixtureClassifier
```

각 클래스가 다른 클래스의 인스턴스를 `self.변수` 로 들고 있습니다.
이것을 **합성(Composition)** 이라고 합니다.
