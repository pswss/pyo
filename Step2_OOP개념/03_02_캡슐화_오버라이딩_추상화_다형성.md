## 1. 캡슐화 (Encapsulation)

**데이터(속성)와 그 데이터를 다루는 동작(메서드)을 하나의 단위로 묶는 것.**

### 캡슐화가 없으면

```python
# 데이터와 처리가 파일 어디에나 흩어져 있음
battery_level = 100.0
motor_speed   = 0.0

def consume_battery(wh):
    global battery_level          # battery_level 을 바꾸는지 코드를 다 읽기 전까진 모름
    battery_level -= wh

def set_speed(v):
    global motor_speed
    motor_speed = max(0.0, min(1.0, v))
```

### 캡슐화가 있으면

```python
class DriveUnit:
    """배터리와 모터를 하나로 묶은 구동 유닛."""

    def __init__(self, capacity_wh: float):
        self._capacity = capacity_wh   # 데이터
        self._charge   = capacity_wh
        self._speed    = 0.0

    def consume(self, wh: float):      # 동작 — 데이터와 함께 있음
        self._charge = max(0.0, self._charge - wh)

    def set_speed(self, v: float):
        self._speed = max(0.0, min(1.0, v))

    @property
    def battery_percent(self) -> float:
        return self._charge / self._capacity * 100
```

### 캡슐화와 은닉화의 차이

| | 캡슐화 | 은닉화 |
|---|---|---|
| 목적 | 관련된 것을 **묶기** | 내부 구현을 **숨기기** |
| 수단 | 클래스 | `_변수`, `__변수`, `@property` |
| 관계 | 은닉화는 캡슐화의 **결과** 중 하나 |

### 프로젝트 예시

`src/robot/robot.py` — `Robot` 클래스:
- **데이터**: `time_step`, `position`, `orientation`, `__time`, `__start_time`
- **동작**: `update()`, `move_wheels()`, `rotate_to_angle()`, `do_loop()`

외부(executor.py)는 `robot.position` 처럼 정해진 인터페이스로만 접근하고,
내부 구현(`pose_manager`, `drive_base` 등)을 알 필요가 없습니다.

## 2. 오버라이딩 심화 (Advanced Overriding)

03회차에서 기본 오버라이딩을 배웠습니다. 여기서는 `super()` 로 **부모 동작을 유지하면서 확장**하는 패턴을 다룹니다.

### super() — 부모 메서드를 호출하면서 추가

```python
class Logger:
    def log(self, msg: str):
        print(f"[LOG] {msg}")

class TimestampLogger(Logger):
    def __init__(self):
        self._tick = 0

    def log(self, msg: str):           # 오버라이딩
        self._tick += 1
        super().log(f"[t={self._tick}] {msg}")   # 부모 log 를 활용해서 확장
```

```python
ts = TimestampLogger()
ts.log("시작")    # [LOG] [t=1] 시작
ts.log("탐색")    # [LOG] [t=2] 탐색
```

### 여러 단계 상속에서의 super()

```python
class FilteredLogger(TimestampLogger):
    def __init__(self, ignore: str):
        super().__init__()          # TimestampLogger.__init__ 실행
        self._ignore = ignore

    def log(self, msg: str):        # 2단계 오버라이딩
        if self._ignore in msg:
            return                  # 차단
        super().log(msg)            # TimestampLogger.log → Logger.log 순으로 실행
```

```
FilteredLogger.log()
  └─ 필터 통과 시 super().log()
        └─ TimestampLogger.log()  → 타임스탬프 추가
              └─ super().log()
                    └─ Logger.log()  → 실제 출력
```

### super() 없이 쓰면 생기는 문제

```python
class BadLogger(TimestampLogger):
    def log(self, msg: str):
        print(f"[LOG] {msg}")   # super() 없이 새로 작성
        # → TimestampLogger 의 타임스탬프 기능이 사라짐
        # → 부모 클래스를 고칠 때 자식이 자동으로 반영되지 않음
```

## 3. 오버로딩 (Overloading)

**같은 이름의 메서드를 인자 타입·개수에 따라 다르게 동작시키는 것.**

Java·C++에서는 언어 차원에서 지원하지만, **Python은 직접 지원하지 않습니다.**
대신 세 가지 방법으로 유사하게 구현합니다.

### 방법 A: 기본값 (Default Arguments)

```python
class Motor:
    def move(self, speed: float = 1.0, duration: float = None):
        if duration is None:
            print(f"속도 {speed} 로 계속 이동")    # move(0.5)
        else:
            print(f"속도 {speed} 로 {duration}초 이동")  # move(0.5, 2.0)
```

### 방법 B: isinstance() 로 타입 분기

```python
class Position:
    def distance_to(self, other) -> float:
        if isinstance(other, tuple):       # distance_to((0.3, 0.4))
            ox, oy = other
        elif isinstance(other, Position):  # distance_to(pos2)
            ox, oy = other.x, other.y
        else:
            raise TypeError(f"지원하지 않는 타입: {type(other)}")
        return math.sqrt((self.x - ox)**2 + (self.y - oy)**2)
```

### 방법 C: *args 가변 인자

```python
class Vector2D:
    def __init__(self, *args, angle_deg: float = None):
        if angle_deg is not None:                         # Vector2D(angle_deg=90)
            rad = math.radians(angle_deg)
            self.x, self.y = math.cos(rad), math.sin(rad)
        elif len(args) == 2:                              # Vector2D(1.0, 0.0)
            self.x, self.y = float(args[0]), float(args[1])
        elif len(args) == 1 and isinstance(args[0], tuple):  # Vector2D((1.0, 0.0))
            self.x, self.y = float(args[0][0]), float(args[0][1])
```

### 오버로딩 vs 오버라이딩

| | 오버로딩 (Overloading) | 오버라이딩 (Overriding) |
|---|---|---|
| 위치 | **같은** 클래스 안 | **상속** 관계 사이 |
| 메서드 이름 | 같음 | 같음 |
| 차이 | **인자**가 다름 | **클래스**가 다름 |
| Python 지원 | 직접 구현 필요 | 기본 지원 |

### 프로젝트 예시

`src/mapping/mapper.py` — `update()`:
```python

           lidar_detections=None, camera_images=None,
           robot_position=None, robot_orientation=None, time=None):
    # 인자가 None 이면 해당 처리 건너뜀

```

## 4. 추상화 (Abstraction)

**공통 인터페이스(메서드 이름과 역할)만 정의하고, 구체적인 구현은 자식 클래스에게 맡기는 것.**

### NotImplementedError 방식 (프로젝트에서 사용하는 방식)

```python
class SubagentInterface:            # src/agent/agent_interface.py
    def update(self, force=False):
        raise NotImplementedError   # "반드시 구현하라" 는 신호

    def get_target_position(self):
        raise NotImplementedError
```

→ 자식이 구현 안 하면 **호출 시점**에 `NotImplementedError` 발생

### abc 모듈 방식 (Python 표준)

```python
from abc import ABC, abstractmethod

class Sensor(ABC):
    @abstractmethod
    def read(self) -> float:    # 자식에서 반드시 구현
        pass

    @abstractmethod
    def unit(self) -> str:
        pass

    def status(self):           # 공통 구현은 여기에 (추상 아님)
        return f"{self.read():.3f} {self.unit()}"
```

```python
s = Sensor("직접")      # → TypeError: 추상 메서드 미구현
                         # NotImplementedError 와 달리 생성 시점에 바로 막힘
```

### 두 방식 비교

| | `raise NotImplementedError` | `ABC + @abstractmethod` |
|---|---|---|
| 오류 발생 시점 | 메서드 **호출** 시 | 인스턴스 **생성** 시 |
| 강제력 | 약함 (생성은 됨) | 강함 (생성 자체가 막힘) |
| 용도 | 간단한 내부 인터페이스 | 공개 API, 라이브러리 |

### 프로젝트 예시

`src/agent/agent_interface.py`:
```python
class SubagentInterface:
    def update(self, force_calculation=False) -> None:
        raise NotImplementedError

    def get_target_position(self):
        raise NotImplementedError

    def target_position_exists(self) -> bool:
        raise NotImplementedError
```

이것을 상속한 클래스들:
- `GoToNonDiscoveredAgent` — 미탐색 지역으로 이동
- `FollowWallsAgent` — 벽을 따라 이동
- `GoToFixturesAgent` — 조난자 위치로 이동

세 클래스 모두 `update()`, `get_target_position()`, `target_position_exists()` 를 구현합니다.

## 5. 다형성 (Polymorphism)

**같은 인터페이스(메서드 이름)를 호출했을 때, 객체의 실제 타입에 따라 다른 동작이 실행되는 것.**

오버라이딩 + 상속이 만들어내는 결과입니다. **"같은 코드 → 다른 결과"**

### 다형성 없이

```python
# 타입마다 분기를 직접 작성해야 함
for strategy in strategies:
    if isinstance(strategy, ConsoleReport):
        strategy.console_report(letter, pos)
    elif isinstance(strategy, FileReport):
        strategy.file_report(letter, pos)
    elif isinstance(strategy, NetworkReport):
        strategy.network_report(letter, pos)
    # 새 타입이 생길 때마다 여기를 고쳐야 함
```

### 다형성 있으면

```python
class ReportStrategy(ABC):
    @abstractmethod
    def report(self, letter: str, position: tuple) -> str:
        pass

class ConsoleReport(ReportStrategy):
    def report(self, letter, position):
        print(f"[콘솔] {letter} @ {position}")

class FileReport(ReportStrategy):
    def report(self, letter, position):
        print(f"[파일] {letter},{position[0]},{position[1]}")

class NetworkReport(ReportStrategy):
    def report(self, letter, position):
        print(f"[네트워크] POST letter={letter}")
```

```python
strategies = [ConsoleReport(), FileReport("log.csv"), NetworkReport("10.0.0.1")]

for s in strategies:
    s.report("H", (0.24, -0.12))   # ← 같은 코드, 다른 동작
    # 새 전략을 추가해도 이 루프는 바꿀 필요 없음
```

### 프로젝트 예시

`src/agent/agent.py`:
```python
self.__navigation_agent = SubagentPriorityCombiner([
    GoToFixturesAgent(mapper),       # update() → 조난자 방향 계산
    FollowWallsAgent(mapper),        # update() → 벽 추적 계산
    GoToNonDiscoveredAgent(mapper),  # update() → 미탐색 BFS 계산
])
```

`SubagentPriorityCombiner.update()` 내부:
```python

    agent.update()           # 같은 update() 호출
    if agent.target_position_exists():
        return agent.get_target_position()
# → 각 에이전트가 어떻게 목표를 계산하는지는 몰라도 됨
```

### 다형성의 핵심 이점

새로운 에이전트 타입을 추가할 때 `SubagentPriorityCombiner` 코드를 **수정할 필요 없음**.
`SubagentInterface` 를 구현하는 새 클래스를 만들어 리스트에 추가만 하면 됩니다.

## 6. 개념 전체 관계

```
클래스
  │
  ├── 캡슐화   : 데이터 + 동작을 하나의 단위로 묶음
  │     └── 은닉화: _변수, __변수, @property 로 내부를 보호
  │
  ├── 상속     : class 자식(부모)
  │     ├── 오버라이딩: 자식이 부모 메서드를 재정의
  │     │     └── super(): 부모 동작을 유지하면서 확장
  │     └── 오버로딩: 같은 이름, 다른 인자 (Python은 직접 구현)
  │
  ├── 추상화   : 인터페이스만 정의, 구현은 자식에게
  │     └── ABC + @abstractmethod (또는 raise NotImplementedError)
  │
  └── 다형성   : 같은 코드가 객체 종류마다 다르게 동작
                  (오버라이딩 + 상속의 결과)
```
